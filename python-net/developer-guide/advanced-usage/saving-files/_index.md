GroupDocs.Metadata provides several ways to save a file after you edit or remove metadata. Saving a file requires a license.

Please refer to the following guides for all overloads of the `save` method:

- [Save a modified file to a specified location]({{< ref "metadata/python-net/developer-guide/advanced-usage/saving-files/save-a-modified-file-to-a-specified-location.md" >}}) — write to a new path.
- [Save a modified file to a stream]({{< ref "metadata/python-net/developer-guide/advanced-usage/saving-files/save-a-modified-file-to-a-stream.md" >}}) — write to a writable binary stream such as `io.BytesIO`.
- [Save a modified file to the original source]({{< ref "metadata/python-net/developer-guide/advanced-usage/saving-files/save-a-modified-file-to-the-original-source.md" >}}) — call `save()` with no argument to write back in place.
